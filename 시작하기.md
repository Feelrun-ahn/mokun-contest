# 목운중 교복 디자인 공모전 사이트 — 올리는 방법

PC를 켜둘 필요 없이 **완전 무료**로 인터넷에 공개합니다.
사이트 화면은 **GitHub Pages**, 데이터·파일 저장은 **Supabase**가 맡습니다.

전체 30~40분 정도 걸립니다. 순서대로만 따라오세요.

---

## 1단계 · Supabase 프로젝트 만들기 (10분)

1. https://supabase.com 접속 → **Start your project** → GitHub 계정으로 로그인
2. **New project** 클릭
   - Name: `mokun-contest`
   - Database Password: 아무거나 만들고 **어딘가에 적어두세요** (다시 못 봅니다)
   - Region: **Northeast Asia (Seoul)** 선택
3. **Create new project** → 1~2분 기다리면 준비 완료

## 2단계 · 데이터베이스 설치 (3분)

1. 왼쪽 메뉴 **SQL Editor** → **New query**
2. 이 폴더의 `setup.sql` 파일을 메모장으로 열어 **전체 복사**
3. 붙여넣고 오른쪽 아래 **Run** 클릭
4. `Success. No rows returned` 가 나오면 성공입니다

> 이 한 번으로 테이블, 보안 규칙, 접수번호 자동발급, 파일 저장소, 공지 초기 내용까지 전부 만들어집니다.

## 3단계 · 학생회 관리자 계정 만들기 (3분)

1. 왼쪽 메뉴 **Authentication** → **Users** → 오른쪽 위 **Add user** → **Create new user**
   - Email: 학생회 이메일 (예: `council@mokun.kr`)
   - Password: 원하는 비밀번호
   - `Auto Confirm User` 체크박스가 보이면 켜주세요. **안 보여도 그냥 넘어가세요.** 아래 2번에서 해결됩니다.
   - **Create user** 클릭

2. 왼쪽 메뉴 **SQL Editor** → **New query** → 아래를 붙여넣고, **이메일 두 곳만** 1번에서 쓴 주소로 바꿔 **Run**

```sql
-- ① 이메일 인증을 완료 처리 (Auto Confirm 체크박스를 못 찾았어도 이걸로 해결됩니다)
update auth.users
   set email_confirmed_at = coalesce(email_confirmed_at, now())
 where email = 'council@mokun.kr';

-- ② 관리자 명단에 등록
insert into admins (user_id, email)
select id, email from auth.users where email = 'council@mokun.kr'
on conflict (user_id) do nothing;

-- ③ 잘 됐는지 확인 (결과에 이메일이 1줄 나오면 성공)
select email from admins;
```

맨 아래에 이메일이 한 줄 나오면 관리자 등록 완료입니다. **User UID를 복사할 필요는 없습니다.**

> ⚠️ **중요**: 이 `admins` 명단에 들어간 계정만 관리자입니다.
> 누가 몰래 회원가입을 해도 작품이나 개인정보는 절대 볼 수 없습니다.
>
> 학생회 임원을 더 추가하려면, 1번처럼 계정을 만든 뒤 위 SQL의 이메일만 바꿔 다시 Run 하면 됩니다.

**추가 잠금 (권장)**: Authentication → Sign In / Providers → Email 에서
**Allow new users to sign up** 을 **꺼두세요.** 아무도 새로 가입할 수 없게 됩니다.

## 4단계 · 열쇠 2개를 사이트에 넣기 (2분)

1. Supabase 왼쪽 아래 **Project Settings** → **API** (또는 Data API)
2. 두 가지를 복사합니다
   - **Project URL** (예: `https://abcdefg.supabase.co`)
   - **anon public** key (아주 긴 문자열)
3. 이 폴더의 `config.js` 를 메모장으로 열어 두 줄을 바꿔 저장합니다

```js
const SUPABASE_URL = 'https://abcdefg.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOi... (아주 긴 문자열)';
```

> `anon public` 키는 공개돼도 괜찮은 키입니다. 보안은 2단계에서 넣은 규칙이 서버에서 막아줍니다.
> **`service_role` 키는 절대 넣지 마세요.** 그건 모든 걸 뚫는 마스터키입니다.

## 5단계 · GitHub에 올리고 공개하기 (10분)

1. https://github.com/new 에서 새 저장소 만들기
   - Repository name: `mokun-contest`
   - **Public** 선택 → **Create repository**
2. 다음 화면에서 **uploading an existing file** 클릭
3. 이 폴더 안의 **파일을 전부 선택(Ctrl+A)해서 올립니다** — 폴더가 없어서 그냥 다 올리면 됩니다
4. 아래 **Commit changes** 클릭
5. 저장소 상단 **Settings** → 왼쪽 **Pages**
   - Source: **Deploy from a branch**
   - Branch: **main** / **/ (root)** → **Save**
6. 1~2분 뒤 새로고침하면 주소가 나옵니다

```
https://<내-깃허브-아이디>.github.io/mokun-contest/
```

이 주소를 학생들에게 공유하면 끝입니다. 🎉

- 참가자 사이트: `https://.../mokun-contest/`
- 학생회 관리자: `https://.../mokun-contest/admin.html`

## 공개하기 전 마지막 점검

- [ ] 관리자 페이지에 **학생회 계정으로만** 로그인되는지 확인
- [ ] 관리자 → **공모전 설정** 에서 제목·접수 마감일·결과 발표일·시상 내역을 실제 내용으로 수정
- [ ] 관리자 → **공지사항 편집** 에서 주의사항·심사기준을 실제 내용으로 수정
- [ ] `mokun-logo.svg` 를 **진짜 학교 로고**로 교체 (지금은 임시 로고입니다)
- [ ] 본인이 직접 테스트 작품을 하나 제출해보고, 관리자에서 다운로드까지 되는지 확인
- [ ] 테스트 작품은 관리자에서 삭제

## 나중에 내용을 바꾸고 싶다면

- **공모전 정보·공지·FAQ·시상 내역** → 관리자 페이지에서 바로 수정 (GitHub 안 건드려도 됨)
- **사이트 디자인·문구** → GitHub에서 파일 수정 → 1~2분 뒤 자동 반영

## 알아두면 좋은 것

- **무료 용량**: 파일 저장 1GB, 파일 하나당 최대 50MB → 교복 디자인 공모전엔 충분합니다.
- **Supabase 휴면**: 7일 동안 아무도 접속하지 않으면 프로젝트가 일시 정지됩니다.
  공모전 기간에는 접속이 있으니 괜찮고, 정지되면 Supabase 대시보드에서 **Restore** 한 번 누르면 됩니다.
- **제출 마감**: 마감일이 지나면 자동으로 제출이 막힙니다. 관리자에서 상태를 '심사 중'으로 바꾸면 확실합니다.
- **개인정보**: 학번·연락처는 학생회 관리자만 볼 수 있습니다. 공모전이 끝나면 관리자에서 작품을 삭제해 파기하세요.

## 문제가 생기면

| 증상 | 원인 |
|---|---|
| 노란 경고 배너가 뜸 | `config.js` 에 URL·키를 안 넣었습니다 (4단계) |
| "데이터를 불러오지 못했습니다" | SQL을 실행 안 했거나(2단계), 키를 잘못 넣었습니다 |
| 관리자 로그인이 안 됨 | 3단계 2번 SQL을 안 돌렸습니다. 이메일 인증이 안 된 계정은 로그인이 막힙니다 |
| "관리자 권한이 없는 계정입니다" | 3단계 2번 SQL의 ②가 안 됐습니다. `select email from admins;` 로 확인해 보세요 |
| 파일 업로드 실패 | SQL의 스토리지 부분이 실행 안 됐습니다. setup.sql을 다시 Run 하세요 |
